# Reglas de Desarrollo

## Pinia
- No duplicar propiedades.
- No usar selectedId para lógica principal.
- Métodos completos siempre entregados.

## Vue
- Componentes desacoplados.
- Props claras.
- No lógica compleja en template.

## Canvas
- Nunca mutar directamente Konva nodes.
- El estado es la fuente de verdad.
- Konva refleja el store, no al revés.

## Entregables
Siempre entregar:
- Métodos completos
- Código listo para copiar/pegar
- Sin fragmentos incompletos
